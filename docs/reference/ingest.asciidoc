[[ingest]]
= Ingest 节点

[partintro]
--
Ingest 节点用于文档被索引前的预处理操作。Ingest 节点解析 bulk 和 index 请求，做相关的数据转换之后再将文档传递给 bulk 或者 index 接口处理。

所有的节点默认都是 ingest 节点，因此任何节点都可以处理 ingest 任务。你也可以创建专用的 ingest 节点。关闭节点的 ingest 功能，可以在 elasticsearch.yml 文件中做如下设定：

[source,yaml]
--------------------------------------------------
node.ingest: false
--------------------------------------------------

在索引数据之前预处理文档，<<pipeline, 定义管道>>章节中定义了一系列的 <<ingest-processors, 处理器>>。每个处理器可以按照一定的方式转换文档。例如，一个管道可以有一个处理器删除文档中的某个字段，紧接着有一个处理器修改一个字段的名称。<<cluster-state, 集群元数据>>会保存所有配置好的管道信息。 

使用管道时，可以简单的在 index 或者 bulk 请求中指定 `pipeline` 参数。这样 ingest 节点才知道使用哪一个管道。例如：

[source,js]
--------------------------------------------------
PUT my-index/_doc/my-id?pipeline=my_pipeline_id
{
  "foo": "bar"
}
--------------------------------------------------
// CONSOLE
// TEST[catch:bad_request]

一个索引如果定义了 <<dynamic-index-settings, 默认管道>>，在 `pipeline` 参数没有指定时将会被使用。

查看 <<ingest-apis,Ingest 接口文档>>获取更多关于创建、添加和删除管道的信息。
